OpenAI's Batch API is the reference "shape" Batchwork is built around: requests are serialized to a JSONL file, uploaded via the Files API, and referenced by a batch you poll until it finishes ‚Äî then the output and error files are downloaded and parsed. Batchwork does all of that for you.

| Property | Value |
| --- | --- |
| **Shape** | JSONL file upload ‚Üí create batch |
| **Endpoints** | `/v1/chat/completions`, `/v1/responses`, `/v1/completions`, `/v1/embeddings`, `/v1/images/generations`, `/v1/images/edits`, `/v1/moderations` |
| **Results** | Output + error files (JSONL) |
| **Webhooks** | Native (`batch.completed`, `batch.failed`, ‚Ä¶) |
| **Env var** | `OPENAI_API_KEY` |
| **Package** | `@ai-sdk/openai` |
| **Base URL** | `https://api.openai.com/v1` |

## Example

```ts
import { batch } from "batchwork";
import { openai } from "@ai-sdk/openai";

const job = await batch({
  model: openai.chat("gpt-5.5"),
  requests: [
    { customId: "fr", prompt: "Capital of France? One word." },
    { customId: "jp", prompt: "Capital of Japan? One word." },
  ],
});
```

## How it works

1. **Build** ‚Äî each request becomes a JSONL line: `{ custom_id, method: "POST", url, body }`.
2. **Upload** ‚Äî the JSONL is uploaded to the Files API with `purpose=batch`.
3. **Create** ‚Äî `POST /v1/batches` references the uploaded file with a 24h `completion_window`.
4. **Poll** ‚Äî Batchwork reads the `status` field until the batch reaches a terminal state.
5. **Read** ‚Äî the `output_file_id` and `error_file_id` are streamed and parsed into [`BatchResult`](/docs/results)s.

Batchwork validates provider-returned batch and file ids before using them in OpenAI-compatible API paths.

## Request shapes

OpenAI exposes several endpoints, and Batchwork mirrors whichever your model implies:

| Model                                   | Batch endpoint           |
| --------------------------------------- | ------------------------ |
| `openai.chat("‚Ä¶")`                      | `/v1/chat/completions`   |
| `openai("‚Ä¶")` / `openai.responses("‚Ä¶")` | `/v1/responses`          |
| `openai.completion("‚Ä¶")`                | `/v1/completions`        |
| `openai.embeddingModel("‚Ä¶")`            | `/v1/embeddings`         |
| `openai.image("‚Ä¶")`                     | `/v1/images/generations` |
| `"openai/omni-moderation-‚Ä¶"` string     | `/v1/moderations`        |

A `"openai/‚Ä¶"` string defaults to chat completions ‚Äî see [Models](/docs/models). The last three rows are submitted via [`batch.embeddings()`](/docs/embeddings), [`batch.images()`](/docs/images), and [`batch.moderations()`](/docs/moderations) rather than `batch()`.

## Embeddings

OpenAI text-embedding models (e.g. `text-embedding-3-small`) batch through `/v1/embeddings` ‚Äî the same upload ‚Üí create ‚Üí poll ‚Üí read flow as chat. Use [`batch.embeddings()`](/docs/embeddings):

```ts
import { batch } from "batchwork";
import { openai } from "@ai-sdk/openai";

const job = await batch.embeddings({
  model: openai.embeddingModel("text-embedding-3-small"),
  requests: [{ customId: "a", value: "The quick brown fox." }],
});
```

Each request embeds one `value`, and the vector lands on `result.embedding`. Shorten it with `providerOptions.openai.dimensions` per request ‚Äî see [Embeddings](/docs/embeddings).

## Images

OpenAI image models (e.g. `gpt-image-2`) batch through `/v1/images/generations` ‚Äî same upload ‚Üí create ‚Üí poll ‚Üí read flow as chat, just a different per-line endpoint. Use [`batch.images()`](/docs/images):

```ts
import { batch } from "batchwork";
import { openai } from "@ai-sdk/openai";

const job = await batch.images({
  model: openai.image("gpt-image-2"),
  requests: [{ customId: "a", prompt: "A red bicycle against a brick wall." }],
});
```

Results come back as inline base64 on `result.images[].data` (with `mediaType`). Pass `size`, `n`, or `providerOptions.openai` (e.g. `quality`) per request ‚Äî see [Images](/docs/images).

OpenAI also batches **image edits** through `/v1/images/edits` ‚Äî source images and masks are passed as JSON references (uploaded `fileId`s or hosted `imageUrl`s), never raw uploads. Use [`batch.images.edit()`](/docs/images#editing-images):

```ts
const job = await batch.images.edit({
  model: openai.image("gpt-image-1.5"),
  requests: [
    {
      customId: "a",
      prompt: "Make the bicycle blue.",
      images: [{ imageUrl: "https://example.com/bicycle.png" }],
    },
  ],
});
```

## Moderations

OpenAI's omni moderation models (e.g. `omni-moderation-latest`) batch through `/v1/moderations` ‚Äî text and/or image inputs per request. The AI SDK has no moderation model type, so pass the `"provider/model"` string form to [`batch.moderations()`](/docs/moderations):

```ts
import { batch } from "batchwork";

const job = await batch.moderations({
  model: "openai/omni-moderation-latest",
  requests: [
    { customId: "a", value: "‚Ä¶user-generated content‚Ä¶" },
    { customId: "b", imageUrls: ["https://example.com/upload.png"] },
  ],
});
```

The verdict lands on `result.moderation` (`flagged`, `categories`, `categoryScores`). Note the batch endpoint list does **not** include `/v1/audio/*` ‚Äî see [Transcriptions](/docs/transcriptions) for the providers that batch audio.

## Webhooks

OpenAI is the only provider with **native** batch webhooks. Mount the handler from the [server layer](/docs/server) to deliver the instant a batch completes ‚Äî no `tick()` polling needed for OpenAI batches.

## Credentials

Set `OPENAI_API_KEY`, or pass `apiKey` / `baseURL` / `headers` to `batch()` to override per call. Install the peer dependency:

```bash
bun add @ai-sdk/openai
```
