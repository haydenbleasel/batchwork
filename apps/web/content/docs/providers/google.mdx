title: Google Gemini
description: Run batches on the Gemini Developer API ‚Äî inline :batchGenerateContent with results keyed by request.

Google's Gemini **Developer API** runs batches inline: requests are submitted to `:batchGenerateContent`, which returns a long-running operation you poll, then reads the responses back inline ‚Äî each correlated by a key you set per request. Batchwork wraps that in the standard `batch()` call.

| Property     | Value                                                |
| ------------ | ---------------------------------------------------- |
| **Shape**    | Inline `:batchGenerateContent`                       |
| **Results**  | Inline operation responses                           |
| **Webhooks** | Poll-only (managed by the server)                    |
| **Env var**  | `GOOGLE_GENERATIVE_AI_API_KEY` (or `GEMINI_API_KEY`) |
| **Package**  | `@ai-sdk/google`                                     |
| **Base URL** | `https://generativelanguage.googleapis.com/v1beta`   |

## Example

```ts
import { batch } from "batchwork";
import { google } from "@ai-sdk/google";

const job = await batch({
  model: google("gemini-2.5-flash"),
  requests: [
    { customId: "fr", prompt: "Capital of France? One word." },
    { customId: "jp", prompt: "Capital of Japan? One word." },
  ],
});
```

A `"google/‚Ä¶"` or `"gemini/‚Ä¶"` model string works too ‚Äî see [Models](/models).

## How it works

1. **Build** ‚Äî each request becomes `{ metadata: { key: customId }, request }`, where `request` is the captured body.
2. **Submit** ‚Äî `POST /v1beta/models/{model}:batchGenerateContent`. The model is set in the URL.
3. **Poll** ‚Äî the returned long-running operation is polled; its `JOB_STATE_*` value maps onto the normalized status.
4. **Read** ‚Äî the inline responses are read back, each correlated by its `metadata.key`.

Batchwork validates Gemini operation ids before using them in API paths.

## Notes

<Callout type="warn">
  Batchwork reads Gemini's **inline** response mode. If a batch returns
  file-mode results (which Gemini may use for very large batches), Batchwork
  throws rather than silently dropping results.
</Callout>

This is the Gemini **Developer API**, not **Vertex AI** ‚Äî Vertex's batch API requires GCS staging and cloud IAM auth, which aren't supported yet. Gemini has no native batch webhook, so it is poll-only; use the [server layer](/server) for unified completion events.

## Images

Gemini **image** models (e.g. `gemini-2.5-flash-image`) run through the same `:batchGenerateContent` flow ‚Äî submit with [`batchImages()`](/images):

```ts
import { batchImages } from "batchwork";
import { google } from "@ai-sdk/google";

const job = await batchImages({
  model: google.imageModel("gemini-2.5-flash-image"),
  requests: [
    { customId: "a", prompt: "A sunlit forest path.", aspectRatio: "16:9" },
  ],
});
```

Images are read back inline from each response's `candidates[].content.parts[].inlineData` and land as base64 on `result.images[].data` (with `mediaType`). Google's **Imagen** models (`:predict`) aren't batch-supported. See [Images](/images).

## Credentials

Set `GOOGLE_GENERATIVE_AI_API_KEY` (or `GEMINI_API_KEY`), or pass `apiKey` / `baseURL` / `headers` to `batch()`. Install the peer dependency:

```bash
bun add @ai-sdk/google
```
